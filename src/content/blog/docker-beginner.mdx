## Docker가 필요한 이유

"내 컴퓨터에서는 되는데..."라는 말을 없애기 위해 Docker가 탄생했습니다. 컨테이너를 통해 개발, 테스트, 프로덕션 환경을 동일하게 유지할 수 있습니다.

## 핵심 개념

- **이미지(Image)**: 컨테이너를 생성하기 위한 읽기 전용 템플릿
- **컨테이너(Container)**: 이미지를 실행한 인스턴스
- **Dockerfile**: 이미지를 빌드하기 위한 명령어 집합

## 첫 번째 Dockerfile

```dockerfile
# Node.js 앱 예시
FROM node:20-alpine

WORKDIR /app

COPY package*.json ./
RUN npm ci --production

COPY . .

EXPOSE 3000
CMD ["node", "server.js"]
```

## 자주 쓰는 명령어

```bash
# 이미지 빌드
docker build -t my-app .

# 컨테이너 실행
docker run -p 3000:3000 my-app

# 실행 중인 컨테이너 확인
docker ps

# 컨테이너 중지
docker stop <container-id>

# 컨테이너 로그 확인
docker logs <container-id>
```

## Docker Compose로 여러 서비스 관리

```yaml
# docker-compose.yml
version: '3.8'
services:
  app:
    build: .
    ports:
      - "3000:3000"
    environment:
      - DATABASE_URL=postgres://user:pass@db:5432/mydb
    depends_on:
      - db

  db:
    image: postgres:15-alpine
    environment:
      POSTGRES_PASSWORD: pass
      POSTGRES_USER: user
      POSTGRES_DB: mydb
    volumes:
      - postgres_data:/var/lib/postgresql/data

volumes:
  postgres_data:
```

## 마치며

Docker는 현대 개발 워크플로우에서 필수적인 도구입니다. 처음엔 복잡해 보이지만, 기본 개념을 익히면 개발 환경 관리가 훨씬 편해집니다.
