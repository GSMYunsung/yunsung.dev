## 개요

사용자가 음식을 촬영하면 자사 AI가 영양소를 분석하고, 영양사가 해당 데이터를 기반으로 맞춤 식단을 제안할 수 있는 앱과 관리 웹을 함께 개발했습니다.

## 주요 작업

- Android / iOS **Code Push 연동**으로 UI 변경 사항을 앱 스토어 배포 없이 즉시 반영되도록 자동화
- 기존 시스템 구조 분석과 요구사항 파악 후, **외부 AI 서비스를 자사 AI 기술로 전환**
- 사용자 설문 페이지 및 관리자 페이지 구조 설계·구현·유지보수
- **React Native 기반 크로스 플랫폼 앱**을 설계부터 구축, QA 리스트 관리 및 지속 유지보수
- 저칼로리 맞춤 영양소 분석 및 환자 식사 기록 뷰어 기능 구현

## 기술적 기여

**필터링 속도 5s → 2.5s 개선**

Remix `action` 함수는 form 내 모든 필드 변경을 일괄 처리합니다. 필터 조건을 `hidden input`으로 관리하던 구조에서, 조건이 바뀔 때마다 검색 로직이 의도치 않게 중복 실행되는 문제가 있었습니다. `sessionStorage`에 이전 검색 상태를 캐싱해 중복 실행을 차단하고, **필터링 응답 속도를 절반으로 단축**했습니다.

**컴포넌트-서버 간 통신 구조 정립**

자식 컴포넌트에서 `action` 없이 Prisma와 직접 통신해야 하는 케이스가 있었습니다. **Remix routes에 경량 API 파일을 분리하고 `axios`로 요청을 라우팅하는 패턴을 정립**해, 컴포넌트 구조를 손상시키지 않고 서버 접근 문제를 해결했습니다.

**Storybook 기반 컴포넌트 설계 패턴 구축**

앱과 관리 웹을 동시에 개발하면서 동일한 UI 요소가 두 플랫폼에 걸쳐 반복 사용되는 상황이 많았습니다. Storybook을 도입해 버튼, 인풋, 카드 등 핵심 컴포넌트를 독립적으로 개발하고 **각 상태(기본·비활성·에러 등)를 Story로 정의**했습니다. 이를 통해 컴포넌트의 재사용 범위를 명확히 하고, **디자이너·개발자 간 UI 기준을 문서 형태로 공유**할 수 있는 환경을 만들었습니다. 개발 중 발생하는 스타일 회귀도 Storybook에서 시각적으로 바로 확인할 수 있어 **QA 부담이 줄었습니다**.
